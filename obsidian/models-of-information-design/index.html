信息设计领域中常用的模型

信息设计理论不仅帮助我们理解如何有效传递信息，还为具体应用场景提供了实践指导。以下是信息设计中的一些常用模型

2024-09-29
2 minute read

Information design

信息设计1 是一种旨在有效传达信息以促进理解的实践，本质是信息的高层次转化。它是图形设计领域的一个特定分支，与数据可视化和信息架构密切相关。

摘要

信息设计（Information Design）是一种专注于以清晰、高效的方式呈现信息的设计实践，它超越了仅仅美观或艺术表达的目的。这一领域在 1970 年代作为一个跨学科的研究领域浮现出来，并在 1979 年《信息设计杂志》的出版后得到了巩固。随后，国际信息设计协会（IIID）和信息设计协会（IDA）在 1987 年和 1991 年成立，进一步推动了该领域的发展。信息设计的早期例子包括威廉・普莱费尔（William Playfair）的商业图表、约翰・斯诺（John Snow）的 kolera 地图、查尔斯・约瑟夫・米纳德（Charles Joseph Minard）的拿破仑俄罗斯战役图表，以及韦伯・E・B・杜博依斯（W.E.B. Du Bois）在 1900 年世界博览会上展示的数据可视化作品等。

简洁性是信息设计中的一个主要关注点，目标是清晰和易于理解。简化信息可能意味着减少信息量，但也可能涉及到增加信息以提高清晰度。简化信息设计的五个步骤包括：

  1. 告诉真相
  2. 直接说明要点
  3. 选择合适的工具
  4. 强调重要内容
  5. 保持简洁

这些步骤有助于信息设计师筛选结果，并保持受众的参与度。

信息模型

在信息设计领域，信息模型是用于组织、结构化和呈现信息的框架，帮助设计者将复杂的内容转化为可理解和易于使用的形式。

1. Hierarchical Model

源自20世纪60年代的信息科学和数据库管理。它通过树状结构组织信息，从更高层次的抽象信息逐渐到细化的具体内容。通过顶层（如总概念或类别）和多个子层级（如具体概念、属性或元素）来组织信息。

2. International System of Typographic Picture Education

国际排版图表教育系统2 （ISOTYPE）由 Otto Neurath 在1920年代提出，是一种基于视觉符传递信息的系统，旨在通过简化的图像来展示复杂的社会、经济和统计数据。使用标准化图标和简化的视觉符号来传递信息，减少语言和文化障碍。

3. Semantic Network Model

语义网络模型源自人工智能和认知科学领域，由 Allen Newell 和 Herbert A. Simon 在20世纪50年代提出，后来由 Tim Berners-Lee 扩展为语义网的基础。

语义网络通过 节点 和 连接 描述概念之间的关系。节点表示概念，连接表示概念之间的关联或属性。它是一种非线性的信息组织方式。

4. Information Architecture

信息架构模型最早由 Richard Saul Wurman 在1970年代提出，目的是帮助组织和管理大量信息，特别是在复杂的系统或网站中。

信息架构通过 层级结构、分类、标签 和 元数据 等方式组织信息，以确保信息的可查找性和逻辑性。它通常结合了用户研究、内容分析和用户体验设计。

5. Flowchart Model

流程图最早由 Frank Gilbreth 在1921年提出，用于描述工业工程中的工作流程。后来，流程图模型被广泛应用于信息系统设计中，以直观地展示过程、步骤和决策点。

其它信息模型

  • 香农-韦弗通信模型 该模型由 Claude Shannon 和 Warren Weaver 在1948年提出，是最早的通信理论模型之一。它源于信息论，旨在研究信息从发件人到接收者的传递过程。
  • Cognitive Work Analysis 认知工作分析模型由 Jens Rasmussen 和 Kim Vicente 在20世纪90年代提出，旨在分析复杂环境中的信息流动和用户行为。CWA 尤其关注用户在动态、复杂系统中的认知需求。
  • Mental Models 心智模型理论由心理学家 Kenneth Craik 于1943年提出，后来由 Don Norman 等人进一步发展，应用于人机交互和信息设计领域。心智模型描述了人们如何通过自己的经验和理解，构建对世界的内部模型，从而预测事件和操作系统。
  • Card Sorting 卡片分类法 3 起源于人机交互和信息架构领域，由 Joseph Novak 在1970年代发展，用于探讨知识结构与用户认知之间的关系。

常见的层次结构模型

层次结构模型在不同的领域和场景下都扮演了重要角色，从教育、心理学到信息系统设计和用户体验。每一个模型都提供了一种层次化的框架，用于有效地组织和管理信息，帮助设计者根据需求选择合适的模型，从而提升信息传递的效果。

1. Bloom’s Taxonomy

布鲁姆分类法由教育心理学家Benjamin Bloom于1956年提出，旨在划分教育目标的层次，帮助教师设计教学策略和评估学生的学习。

模型结构

  1. 记忆 (Remembering)：回忆事实和基础信息。
  2. 理解 (Understanding)：解释信息的意义。
  3. 应用 (Applying)：在新情境中使用知识。
  4. 分析 (Analyzing)：分解信息并理解其结构。
  5. 评价 (Evaluating)：根据标准判断信息的价值。
  6. 创造 (Creating)：整合信息以生成新的想法或产品。

2. DIKW Pyramid

DIKW 金字塔 是信息设计领域中一个常见的模型，它代表了数据 (Data)、信息 (Information)、知识 (Knowledge) 和智慧 (Wisdom) 的层级关系。

Russell Ackoff 在其1989年的文章《From Data to Wisdom》中阐述了这个概念，随后这个模型被广泛应用于信息管理、决策支持系统、知识管理等多个领域。

3. Ackoff 系统层次模型

Russell Ackoff 是 DIKW 金字塔背后的关键人物之一，他的系统层次理论强调从简单的结构到复杂的系统进行组织和理解。

模型结构

  1. 数据 (Data)：原始的、无组织的信息。
  2. 信息 (Information)：经过处理和分析的数据。
  3. 知识 (Knowledge)：经过解释和应用的信息。
  4. 理解 (Understanding)：对知识之间关系的深入掌握。
  5. 智慧 (Wisdom)：基于理解的决策和行动。

Ackoff 的系统模型类似于 DIKW 金字塔，但它在“理解”层次上有所延伸，强调知识的深度理解。这一模型在信息系统设计、知识管理和战略规划中被广泛应用。

4. Nielsen’s Usability Hierarchy

由 Jakob Nielsen 提出，用于评估系统或设计的可用性。它是用户体验设计中重要的理论框架。

模型结构

  1. 有用性 (Utility)：设计是否满足用户需求。
  2. 可用性 (Usability)：设计是否易于使用。
  3. 有效性 (Effectiveness)：用户是否能高效地完成任务。
  4. 易学性 (Learnability)：用户是否容易掌握系统的使用。
  5. 可记忆性 (Memorability)：用户是否能在一段时间后轻松记住如何使用系统。

这个模型帮助信息设计者确保系统和界面在设计中具备良好的可用性特性，广泛应用于网站、应用程序和软件界面的设计中。

5. SECI Model

由 Nonaka 和 Takeuchi 在1990年代提出，主要用于描述组织内部知识转化的过程，强调隐性知识和显性知识的交互。

模型结构

  1. 社会化 (Socialization)：隐性知识之间的共享。
  2. 外部化 (Externalization)：将隐性知识转化为显性知识。
  3. 组合 (Combination)：显性知识之间的组合和重组。
  4. 内化 (Internalization)：将显性知识转化为隐性知识，形成个人经验。 该模型常用于知识管理和组织学习中，帮助企业设计知识共享和创新流程。例如，在企业信息系统或培训计划中，SECI 模型帮助设计者组织和促进知识的有效传递。

DIKW pyramid

DIKW 金字塔4 是一种描述数据、信息、知识和智慧之间关系的模型，认为这些元素通过四个质量阶段深入理解一个主题：数据、信息、知识和智慧。

DIKW 金字塔，也称为 DIKW 层次、智慧层次、知识层次、信息层次、信息金字塔和数据金字塔，是一种模型，用于表达数据、信息、知识和智慧之间的结构或功能关系。该模型最初由 T. S. Eliot 的诗句启发，后来被多位学者如 Clarence W. Barron、Kenneth Boulding、Nicholas L. Henry、Yi-Fu Tuan、Daniel Bell、Mike Cooley、Milan Zeleny 和 Russell Ackoff 等进一步发展。模型的每一层都基于下一层，从数据开始，通过信息、知识最终达到智慧。

  • 数据 被视为没有用途的符号或信号
  • 信息 是有用的、组织化的数据
  • 知识 是应用信息的能力
  • 智慧 则是对知识的评估和使用，以做出明智的决策

不过，对于智慧的定义和位置在学术界存在争议，有些模型没有包括智慧这一层次。还有一些其他的变体，会增加一些层次，比如理解。

该模型也受到了一些批评，如

  • 哲学家 Rafael Capurro 认为，数据是一种抽象，信息是“传递意义的行为”，知识则是系统通过交流从“世界”中选择意义的事件，因此，这些概念之间的层次关系是一种误解
  • Zins 质疑 DIKW 模型，认为很难明确区分信息与知识
  • 批评者如 David Weinberger 认为，DIKW金字塔过于简化了数据、信息、知识和智慧之间的相互作用
  • 美国哲学家 John Dewey 和 Arthur Bentley 提出了比 DIKW 模型更复杂的替代方案，包含19个“术语指引”

呈现

图像呈现。DIKW 还也被表示为二维图表或一个或多个流程图。在这种情况下，元素之间的关系可能被呈现为更少的等级制度，包含反馈循环和控制关系。

计算呈现。智能决策支持系统正在尝试通过引入一般模拟领域的新技术和方法，特别是在基于代理的建模的上下文中，从智能软件代理领域引入新技术和方法，以改进决策过程。

Inverted pyramid

倒金字塔 5 是新闻写作中的一种信息组织和优先级划分的方法，它要求重要信息放在文章开头，逐步细化到次要信息，这种形式适用于新闻报道、博客、专栏和市场宣传等多种文本类型。
